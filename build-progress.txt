## Iteration 1 — infra-001: Database Schema
Date: 2026-04-08

### What was built
- Complete Drizzle ORM schema with 10 tables: organizations, org_memberships, projects, deployments, pages, api_keys, agent_jobs, assistant_conversations, analytics_events, audit_logs
- 7 PostgreSQL enums: org_plan, org_role, project_status, deployment_status, api_key_type, agent_job_status, analytics_event_type
- All tables have UUID primary keys, timestamps with timezone, proper foreign keys with cascade deletes
- Unique indexes on org slugs, project slugs per org, page paths per project, user-org membership pairs
- Performance indexes on all foreign key columns and frequently queried fields

### Test results
- 28 unit tests (schema.test.ts) — all passing
- 49 total unit tests — all passing
- make check (typecheck + lint) — passing
- Schema pushed to RDS Postgres via make db-push — success

### Files changed
- src/lib/db/schema.ts — complete schema (was placeholder)
- tests/schema.test.ts — new, 28 tests
- tests/e2e/smoke.spec.ts — new, 2 smoke tests

### Decisions
- Used UUID (defaultRandom) for all primary keys — matches SaaS patterns
- userId in orgMemberships/auditLogs is text (not UUID) to match Better Auth's user.id type
- JSON columns (settings, frontmatter, messages, data, details) typed with $type<> for TypeScript safety
- Cascade deletes on all parent-child relationships (org->projects, project->pages, etc.)

## Iteration 2 — infra-002: S3 File Upload Configuration
Date: 2026-04-08

### What was built
- S3 utility module (`src/lib/s3.ts`) with presigned URL generation for uploads and downloads
- Key builder function following `{orgId}/{projectId}/assets/{filename}` convention
- Upload validation: allowed content types whitelist, path traversal protection, filename validation
- Object lifecycle helpers: delete, exists check
- API route `POST /api/uploads/presign` — generates presigned PUT URL for direct client-to-S3 upload
- API route `GET /api/uploads/presign?key=...` — generates presigned GET URL for downloads
- E2E tests for the presign API endpoint (5 tests)

### Test results
- 16 new unit tests (s3.test.ts) — all passing
- 5 new E2E tests (uploads-api.spec.ts) — written, QA will verify
- 65 total unit tests — all passing
- make check (typecheck + lint) — passing

### Files changed
- src/lib/s3.ts — new, S3 client + presigned URLs + validation
- src/app/api/uploads/presign/route.ts — new, API route for presign
- tests/s3.test.ts — new, 16 unit tests
- tests/e2e/uploads-api.spec.ts — new, 5 E2E tests

### Decisions
- Used allowlist for content types (images, docs, media, fonts) — blocks executables
- 15-minute presigned URL expiry — standard for upload flows
- 50MB max upload size — enforced at validation layer
- Path traversal blocked by rejecting ".." in filenames
- S3 bucket already provisioned by preflight.sh, using env var S3_BUCKET
